📊 Logic Overview
This project demonstrates the fundamental mechanics of **Post-Test Iteration** in C++:

- **Guaranteed Execution:** Unlike a standard `while` loop, the code block inside a `do-while` structure executes at least once before the condition is ever checked.
- **Input Handling:** The program captures a user-defined integer (`num`) to determine the boundary of the loop.
- **Counter Synchronization:** A control variable (`i`) is incremented within the loop to track iterations and eventually trigger the exit condition.

🛠️ Features
- **Deterministic Iteration:** Clearly showcases how the loop behaves when the exit condition is met vs. when it is not.
- **Interactive Input:** Allows real-time testing of loop logic through console input.
- **Clean Documentation:** Code is structured for educational clarity, following professional naming conventions.

🚀 Execution & Environment Guide
To run this project, you will need a C++ compiler (like GCC) or a dedicated IDE.

### 1. Recommended Environment
- **Dev-C++** (Simple and fast for beginners)
- **Visual Studio Code** (Professional development)
- **Online Compilers:** Use [OnlineGDB](https://www.onlinegdb.com/) for quick browser-based testing.

### 2. Simple Execution Procedure
**Step 1: Setup the Project**
- Create a file named `do_while_loop.cpp`.
- Copy and paste the source code into your editor.

**Step 2: Compile the Code**
- **In Dev-C++:** Press **F11** to Compile and Run.
- **In Terminal:** Use `g++ do_while_loop.cpp -o LoopDemo` followed by `./LoopDemo`.

**Step 3: Observe Logic**
- Enter a positive number to see the loop iterate. 
- Enter a negative number or zero to observe the loop's "at-least-once" execution rule.
